# Game Design Document

## Концепция

Idle mining / exploration игра, вдохновлённая Deep Corp и Gold & Goblins.
Игрок исследует подземные уровни, добывает ресурсы, нанимает и объединяет работников,
выполняет заказы, расширяет склады и постепенно спускается глубже.

## Игровой цикл

```
Планирование смены
  → распределить работников по клеткам
  → принять/отклонить заказы
  → купить/улучшить склады, нанять работников

Активная смена (300 тиков / 5 минут)
  → работники добывают ресурсы
  → ресурсы идут на склады
  → прогресс разработки клеток накапливается
  → открытые клетки расширяют область разведки

Конец смены
  → распределение ресурсов по активным заказам
  → автосохранение
  → переход в планирование
```

## Уровни

- Уровни — бесконечные процедурные карты, генерируемые по seed чанками 32×32
- Стартовая область уровня: пустая зона 3×3 в центральном чанке
- Зона спуска на следующий уровень: пустая зона 3×3 в нецентральном чанке (радиус 5×5 чанков от старта)
- Зона спуска скрыта до попадания в радиус разведки 5 клеток
- Чем глубже уровень, тем выше базовая трудоёмкость клеток и богаче ресурсы

### Размер уровней

```
size(depth) = min(baseLevelSize * levelSizeMultiplier^depth, maxLevelSize)
```

Конфиг (balance): `baseLevelSize = 100`, `levelSizeMultiplier = 1.10`, `maxLevelSize = 500`

## Клетки

| Тип           | Описание                                  |
| ------------- | ----------------------------------------- |
| `empty`       | Пустота, проходима, не требует разработки |
| `deposit`     | Порода и/или ресурсы, требует работника   |
| `obstacle`    | Неразрушаемое препятствие                 |
| `stairs_area` | Часть зоны спуска 3×3                     |

Клетка `deposit` содержит компоненты:

```
{ type: "rock" | "resource", resourceId?, ratio, initialAmount, remainingAmount }
```

Порода (rock) всегда добываема, исчезает без попадания на склад.
Ресурс добывается только при наличии места на складе.
При `remainingAmount = 0` всех компонентов клетка становится `empty`.

### Трудоёмкость клетки

```
labor(cell) = baseLabor(depth) * distanceMultiplier(distance) * resourceModifier(cell)
distanceMultiplier(d) = 1.01^d   (compound от расстояния до входа в клетках)
```

## Видимость

- `unknown` — клетка не видна
- `scouted` — клетка видна (тип, компоненты, количество)

Разведка: радиус 5 клеток от каждой достижимой клетки.
При пробуривании к пустой области — вся связная пустота раскрывается сразу.

## Достижимость

`reachable` — от входа уровня есть путь через пустые/открытые клетки.
Только достижимые клетки могут использоваться для размещения работников.
Пересчитывается после каждого открытия клетки (BFS/flood-fill).

## Ресурсы

12 ресурсов, упорядоченных по редкости и минимальной глубине появления:

| #   | Ресурс     | Мин. глубина | Редкость  |
| --- | ---------- | ------------ | --------- |
| 1   | Stone      | 0            | common    |
| 2   | Coal       | 0            | common    |
| 3   | Iron       | 1            | common    |
| 4   | Copper     | 1            | common    |
| 5   | Silver     | 2            | uncommon  |
| 6   | Gold       | 3            | uncommon  |
| 7   | Crystal    | 4            | rare      |
| 8   | Emerald    | 5            | rare      |
| 9   | Ruby       | 6            | rare      |
| 10  | Diamond    | 7            | epic      |
| 11  | Mithril    | 9            | epic      |
| 12  | Adamantite | 11           | legendary |

На каждом новом уровне гарантированно добавляется минимум один новый ресурс в пул генерации.

## Работники

- Глобальный пул игрока (не привязан к уровню)
- Уровни 1–N, скорость разработки растёт с уровнем
- Максимум 4 работника на одну закрытую клетку (по одному с каждой стороны)
- Один работник занимает одну соседнюю открытую достижимую клетку
- Скорость разработки = сумма скоростей назначенных работников

### Покупка

```
maxPurchasableWorkerLevel = max(1, highestUnlockedWorkerLevel - 2)
```

По умолчанию доступен только 1 уровень.

### Merge

```
2 работника уровня N → 1 работник уровня N+1
```

Только для свободных работников. Только в фазе `shift_planning`.

## Склады

- Отдельный склад для каждого типа ресурса
- Вместимость нескольких складов одного типа суммируется
- Покупка и улучшение — только в `shift_planning`
- Если склад ресурса заполнен — добыча этого ресурса приостанавливается

## Заказы

- Требуют один или несколько типов ресурсов
- Ограниченное время доступности (`expiresAtTick`)
- Если ресурсов достаточно — выполняются немедленно
- Если нет — можно принять в работу (нельзя отменить после принятия)
- В конце каждой смены ресурсы распределяются между активными заказами (`priority_based`)
- Вознаграждение: деньги

### Состояния заказа

```
available → accepted → completed
available → declined
available → expired
```
